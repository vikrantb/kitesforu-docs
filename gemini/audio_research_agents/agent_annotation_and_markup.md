# KitesForU Annotation & Markup Framework (KAMF)
**Version:** 1.0 (2026)
**Subject:** Standardizing Natural Performance in AI Speech Synthesis

## 1. The Multi-Layer Synthesis Philosophy
Natural speech at KitesForU is achieved through a three-layer hierarchy. We do not treat audio as "text-to-speech"; we treat it as **"Intent-to-Performance."**

1.  **Semantic Layer (The Script):** The literal words generated by the LLM.
2.  **Annotation Layer (The Director):** Metadata and markup that describe *how* words are delivered.
3.  **Synthesis Layer (The Actor):** The provider-specific execution (ElevenLabs, Gemini, etc.).

## 2. Universal Annotation Schema (UAS)
KitesForU uses a lightweight, provider-agnostic markup called **KAMF-Tags**. These are inserted by the Script-Generator LLM and translated by the **Course Audio Worker** before reaching the TTS provider.

### Core Tag Categories
| Category | Syntax | Usage in KitesForU |
| :--- | :--- | :--- |
| **Pacing** | `<pause: duration>` | `beat`, `short`, `long`. Critical for lecture transitions. |
| **Persona** | `[as: persona_id]` | Applies a consistent vocal profile (e.g., "Professor_M", "Teen_F"). |
| **Emotion** | `(emotion: intensity)` | `warm`, `urgent`, `curious`, `authoritative`. |
| **Vocal-FX** | `*vocal_action*` | `*sighs*`, `*chuckles*`, `*breath*`. Use sparingly (max 1 per 2 sentences). |
| **Emphasis** | `_word_` | Semantic stress for key learning concepts. |

## 3. Provider Mapping Table (2026 Strategy)
The KAMF-Tags are translated dynamically based on the active provider for the course.

| KAMF Tag | ElevenLabs v3 Mapping | OpenAI gpt-4o Mapping | Google Gemini TTS Mapping |
| :--- | :--- | :--- | :--- |
| `(urgent)` | Style Exaggeration: 0.8 | `instructions: "High energy"` | `Director's Note: "Deliver with urgency"` |
| `*sighs*` | Parenthetical: `(sighs)` | Parenthetical: `(sighs)` | Inline: `[audio_action: sigh]` |
| `_concept_` | Stability: 0.3 | Wrapped in `*concept*` | `Context: "Emphasize this word"` |

## 4. KitesForU Naturalness Strategy: "The 3-Tier Approach"

### Tier A: The Authority (Lectures)
*   **Focus:** Clarity with "Human Warmth."
*   **Markup Strategy:** Minimal fillers. Uses `<pause: short>` to simulate thinking before complex explanations. Uses `(warm)` to avoid the "robotic instructor" feel.

### Tier B: The Interaction (Conversational Dialogue)
*   **Focus:** Realism and Friction.
*   **Markup Strategy:** Heavy use of `*breath*`, `*uh*`, and `overlap`. This is used for "Roleplay" scenarios in courses (e.g., practicing a sales call).

### Tier C: The Storyteller (Narrative Content)
*   **Focus:** Immersion and Subtext.
*   **Markup Strategy:** High-intensity emotion tags `(grief-struck)` and long `(silence)` markers.

## 5. Implementation & Validation Protocol

### A. The "Goldilocks" Rule (Automation Logic)
To prevent "over-acting," the Script-Generator must adhere to these density limits:
*   **Maximum:** 1 Performance Tag per sentence.
*   **Minimum:** 1 Performance Tag every 3 sentences (to prevent "the robot bleed").

### B. Phonetic Safety Net
For technical courses (Coding, Medical, etc.), use the `[ipa: ...]` tag for any word not found in the standard dictionary to prevent the TTS from "guessing" and breaking immersion.

### C. Validation (The "Ear Test")
Before a course is finalized, the **Audio QA Worker** runs a "Naturalness Score" (using a secondary LLM) to check:
1.  **Hallucination Check:** Did the TTS read the markup out loud? (e.g., saying the word "sigh" instead of sighing).
2.  **Pacing Check:** Is the total duration within 10% of the predicted "Natural Reading Time"?

## 6. Future-Proofing
As models move toward **Native Multimodal Generation** (where text and audio are generated simultaneously), KAMF will transition from "Tags" to **"Vector Embeddings."** In 2027, KitesForU will likely feed "Vocal Style Vectors" directly into the generator, making manual markup obsolete. This framework acts as the bridge until that point.
