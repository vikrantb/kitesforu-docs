# Second-Pass Brief

This brief is for the next round of product thinking after the first-pass package in this folder has been read.

## Purpose

Do not simply agree with the initial list. Stress-test it.

The real goal is to answer:

> Which ideas would create the most meaningful improvement in the lived user experience of KitesForU, and which ideas sound smart but would not move the product enough?

## What to challenge

Challenge the first-pass package on all of these dimensions:

- what is right
- what is too abstract
- what overstates the problem
- what underestimates implementation cost
- what sounds elegant but would not actually reduce user confusion
- what is missing from the list

## Required thinking mode

Treat this as a hard product problem:

- use `--ultrathink --seq`
- do deeper research where needed, not just surface critique
- push beyond local page polish and think in terms of scenario loops

## Recommended research-agent split

Use at least 5 clearly-scoped research agents:

1. `entry-and-time-to-value`
   Focus on onboarding interruption, join flows, dead ends, and first-value speed.
2. `navigation-naming-and-information-scent`
   Focus on route language, page naming, later-page framing, and comprehension.
3. `creation-confidence-and-clarifier-preemption`
   Focus on input guidance, good defaults, and high-confidence creation without heavy forms.
4. `teacher-student-corporate-workflow-continuity`
   Focus on the end-to-end scenario loops, not isolated screens.
5. `trust-status-and-resume`
   Focus on progress visibility, next-step clarity, and recovery / resumption behavior for long-running AI work.

## Deliverables for the second pass

### 1. Honest critique

Which parts of the first-pass package are strongest, weakest, or too hand-wavy?

### 2. Added ideas

What important improvements are missing?

### 3. Consolidated shortlist

Reduce the full option set down to a smaller set of high-conviction changes.

### 4. Final prioritization

Produce a final ranking of the next UX-improvement feature set KitesForU should actually build.

### 5. Implementation-minded framing

For the top priorities, define the smallest shippable version that would still create a real user-experience change.

## Final standard

The best output from the second pass should not sound like a design review.

It should sound like a product strategy for making the product easier to use in real life.
